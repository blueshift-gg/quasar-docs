title: Metaplex Metadata

The Metaplex Token Metadata program attaches metadata (name, symbol, URI, creators, collections) to SPL Token mints. The `quasar-metadata` package provides zero-copy account types, typed CPI methods, and account behaviors for metadata and master edition accounts.

```toml
[dependencies]
quasar-metadata = "0.1.0"
```

## Account types

### MetadataAccount

`Account<MetadataAccount>` validates that an account is owned by the Metaplex Token Metadata program (`metaqbxxUerdq28cj1RbAWkYQm3ybzjb6a8bt518x1s`), has at least 65 bytes of data, and that the first byte (the Metaplex `Key` discriminant) is `MetadataV1` (4).

It derefs to `MetadataPrefix`, covering the fixed 65-byte header:

```rust
// Fields accessible via zero-copy deref:
metadata.key()               // u8 (always 4 for MetadataV1)
metadata.update_authority()  // &Address
metadata.mint()              // &Address
```

Fields after the prefix (name, symbol, URI, creators, collection, uses) are variable-length Borsh-serialized data. The fixed prefix gives constant-time access to update authority and mint without parsing the entire account.

### MasterEditionAccount

`Account<MasterEditionAccount>` validates ownership by the Metadata program, at least 18 bytes of data, and that the `Key` discriminant is `MasterEditionV2` (6).

It derefs to `MasterEditionPrefix`:

```rust
// Fields accessible via zero-copy deref:
edition.key()              // u8 (always 6 for MasterEditionV2)
edition.supply()           // u64 -- number of editions printed
edition.max_supply_value() // Option<u64> -- None means unlimited
```

## The MetadataProgram type

`Program<MetadataProgram>` validates the program address and provides CPI methods through the `MetadataCpi` trait. Both come from `quasar-metadata`:

```rust
use quasar_metadata::{MetadataCpi, MetadataProgram};

pub metadata_program: Program<MetadataProgram>,
```

## Declarative metadata initialization

Use the `metadata(...)` behavior on `Account<MetadataAccount>` to create and validate a mint's metadata PDA:

```rust
use quasar_metadata::{accounts::metadata, MetadataAccount, MetadataProgram};

#[derive(Accounts)]
pub struct CreateMetadata {
    #[account(mut)]
    pub payer: Signer,
    pub mint: UncheckedAccount,
    pub mint_authority: Signer,
    pub update_authority: Signer,
    pub metadata_program: Program<MetadataProgram>,
    pub system_program: Program<SystemProgram>,
    pub rent: Sysvar<Rent>,
    #[account(
        init,
        payer = payer,
        metadata(
            program = metadata_program,
            mint = mint,
            mint_authority = mint_authority,
            update_authority = update_authority,
            system_program = system_program,
            rent = rent,
            name = "My NFT",
            symbol = "MNFT",
            uri = "https://example.com/nft.json",
            seller_fee_basis_points = 500,
            is_mutable = true
        )
    )]
    pub metadata: Account<MetadataAccount>,
}
```

During account parsing, Quasar calls `create_metadata_accounts_v3` through the supplied Metadata program. For an existing metadata account, `metadata(program = metadata_program, mint = mint)` validates the program, mint, and PDA address without invoking the create CPI.

The Metaplex program derives and allocates the metadata PDA from `["metadata", metadata_program_id, mint_address]`. Quasar loads the result as `Account<MetadataAccount>`.

### Behavior arguments

| Argument | Type | Description |
|----------|------|-------------|
| `program` | account field | Token Metadata program; required for validation and initialization |
| `mint` | account field | Mint described by the metadata; required for validation and initialization |
| `mint_authority` | account field | Current mint authority; required for initialization |
| `update_authority` | account field | Metadata update authority; required for initialization and optional during validation |
| `system_program` | account field | System program passed to the create CPI |
| `rent` | account field | Rent sysvar passed to the create CPI |
| `name` | `&str` | Token name, up to 32 bytes |
| `symbol` | `&str` | Token symbol, up to 10 bytes |
| `uri` | `&str` | Off-chain metadata URI, up to 200 bytes |
| `seller_fee_basis_points` | `u16` | Royalty in basis points; defaults to `0` |
| `is_mutable` | `bool` | Whether metadata can be updated; defaults to `true` |

The CPI builders enforce these maximum lengths and return `ProgramError::InvalidInstructionData` when a field exceeds its limit.

## CPI methods

The `MetadataCpi` trait provides all Metaplex Token Metadata CPI methods.

### Variable-length instructions

These instructions contain variable-length string fields and return `Result<CpiDynamic<...>>`. Propagate the builder result before invoking the CPI.

#### create_metadata_accounts_v3

Creates a metadata account for an existing mint:

```rust
self.metadata_program
    .create_metadata_accounts_v3(
        &self.metadata,
        &self.mint,
        &self.mint_authority,
        &self.payer,
        &self.update_authority,
        &self.system_program,
        &self.rent,
        "Token Name",     // name
        "TKN",            // symbol
        "https://...",    // uri
        500,              // seller_fee_basis_points
        true,             // is_mutable
        true,             // update_authority_is_signer
    )?
    .invoke()?;
```

Create parameters accept `&str` and `&[u8]`. String literals work as written; convert a byte string with `&b"..."[..]`.

#### update_metadata_accounts_v2

Updates an existing metadata account. To replace `DataV2`, pass `Some` for `name`, `symbol`, and `uri`; if any of the three is `None`, the builder omits the `DataV2` update. The authority and boolean options remain independent.

```rust
self.metadata_program
    .update_metadata_accounts_v2(
        &self.metadata,
        &self.update_authority,
        None,                         // new_update_authority
        Some(&b"New Name"[..]),       // name
        Some(&b"NEW"[..]),            // symbol
        Some(&b"https://new"[..]),    // uri
        Some(500),                    // seller_fee_basis_points
        None,                         // primary_sale_happened
        Some(false),                  // is_mutable
    )?
    .invoke()?;
```

### Fixed-length instructions

These use regular `CpiCall` with const-generic sizes.

#### create_master_edition_v3

Creates a master edition account, making the mint a verified NFT:

```rust
self.metadata_program
    .create_master_edition_v3(
        &self.edition,
        &self.mint,
        &self.update_authority,
        &self.mint_authority,
        &self.payer,
        &self.metadata,
        &self.token_program,
        &self.system_program,
        &self.rent,
        Some(0), // max_supply: 0 = unique 1/1, None = unlimited
    )
    .invoke()?;
```

#### mint_new_edition_from_master_edition_via_token

Mints a new edition from an existing master edition:

```rust
self.metadata_program
    .mint_new_edition_from_master_edition_via_token(
        &self.new_metadata,
        &self.new_edition,
        &self.master_edition,
        &self.new_mint,
        &self.edition_mark_pda,
        &self.new_mint_authority,
        &self.payer,
        &self.token_account_owner,
        &self.token_account,
        &self.new_metadata_update_authority,
        &self.metadata,
        &self.token_program,
        &self.system_program,
        &self.rent,
        edition_number, // u64
    )
    .invoke()?;
```

### Collection management

```rust
// Verify a collection item
self.metadata_program.verify_collection(
    &self.metadata, &self.collection_authority, &self.payer,
    &self.collection_mint, &self.collection_metadata,
    &self.collection_master_edition,
).invoke()?;

// Set and verify in one call
self.metadata_program.set_and_verify_collection(
    &self.metadata, &self.collection_authority, &self.payer,
    &self.update_authority, &self.collection_mint,
    &self.collection_metadata, &self.collection_master_edition,
).invoke()?;

// Unverify
self.metadata_program.unverify_collection(
    &self.metadata, &self.collection_authority,
    &self.collection_mint, &self.collection_metadata,
    &self.collection_master_edition,
).invoke()?;

// Set collection size
self.metadata_program.set_collection_size(
    &self.metadata, &self.update_authority, &self.mint, size,
).invoke()?;
```

Sized variants are also available: `verify_sized_collection_item`, `set_and_verify_sized_collection_item`, and `unverify_sized_collection_item`.

### Creator and authority management

```rust
// Sign metadata as a creator
self.metadata_program.sign_metadata(&self.creator, &self.metadata).invoke()?;

// Remove creator verification
self.metadata_program.remove_creator_verification(&self.creator, &self.metadata).invoke()?;

// Approve a collection authority
self.metadata_program.approve_collection_authority(
    &self.record, &self.new_authority, &self.update_authority,
    &self.payer, &self.metadata, &self.mint,
).invoke()?;

// Revoke a collection authority
self.metadata_program.revoke_collection_authority(
    &self.record, &self.delegate, &self.revoke_authority,
    &self.metadata, &self.mint,
).invoke()?;
```

### Token operations

```rust
// Freeze a delegated token account
self.metadata_program.freeze_delegated_account(
    &self.delegate, &self.token_account, &self.edition,
    &self.mint, &self.token_program,
).invoke()?;

// Thaw a delegated token account
self.metadata_program.thaw_delegated_account(
    &self.delegate, &self.token_account, &self.edition,
    &self.mint, &self.token_program,
).invoke()?;

// Burn an NFT
self.metadata_program.burn_nft(
    &self.metadata, &self.owner, &self.mint,
    &self.token, &self.edition, &self.spl_token,
).invoke()?;

// Update primary sale flag
self.metadata_program.update_primary_sale_happened_via_token(
    &self.metadata, &self.owner, &self.token,
).invoke()?;

// Set token standard
self.metadata_program.set_token_standard(
    &self.metadata, &self.update_authority, &self.mint,
).invoke()?;
```

## CpiDynamic vs CpiCall

Most Metaplex CPI methods use `CpiCall` when the instruction data size is known at compile time. `create_metadata_accounts_v3` and `update_metadata_accounts_v2` use `CpiDynamic` with a bounded 512-byte data buffer. Both support `.invoke()` and `.invoke_signed()` after the builder succeeds.

## InitMetadata and InitMasterEdition traits

For manual initialization outside the declarative attribute system, use these traits.

### InitMetadata

```rust
use quasar_metadata::InitMetadata;

self.metadata.init(
    &self.metadata_program,
    &self.mint,
    &self.mint_authority,
    &self.payer,
    &self.update_authority,
    &self.system_program,
    &self.rent,
    b"Token Name",
    b"TKN",
    b"https://example.com/meta.json",
    0,     // seller_fee_basis_points
    true,  // is_mutable
)?;

// Or with PDA signing:
self.metadata.init_signed(
    &self.metadata_program,
    &self.mint,
    &self.mint_authority,
    &self.payer,
    &self.update_authority,
    &self.system_program,
    &self.rent,
    b"Token Name",
    b"TKN",
    b"https://example.com/meta.json",
    0,
    true,
    &seeds,
)?;
```

### InitMasterEdition

```rust
use quasar_metadata::InitMasterEdition;

self.master_edition.init(
    &self.metadata_program,
    &self.mint,
    &self.update_authority,
    &self.mint_authority,
    &self.payer,
    &self.metadata,
    &self.token_program,
    &self.system_program,
    &self.rent,
    Some(0), // max_supply: 0 = unique 1/1 NFT
)?;
```

Both traits also provide `init_signed()` variants that accept PDA seeds for cases where the mint authority or update authority is a PDA.
