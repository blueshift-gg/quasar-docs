## Prerequisites

Install [Rust](https://rustup.rs/) and the [Solana CLI](https://docs.anza.xyz/cli/install):

Quasar v0.1.0 programs require Rust 1.89.0 or newer.

```bash
# Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Solana CLI
sh -c "$(curl -sSfL https://release.anza.xyz/v4.1.1/install)"
```

### Supported development hosts

Quasar v0.1.0 supports Ubuntu 24.04 x86-64, macOS 15 on Apple Silicon, and macOS 15 on Intel. On an x86-64 Windows machine, use WSL2 with Ubuntu 24.04 x86-64.

Native Windows and other operating-system or architecture combinations are not supported by v0.1.0.

## Install the Quasar CLI

```bash
cargo install quasar-cli --version 0.1.0 --locked
```

Verify with `quasar --version`.

## Toolchain

Quasar compiles programs to SBF (Solana Bytecode Format). Two toolchains are available:

| Toolchain | Build command | Notes |
|-----------|---------------|-------|
| **Solana** (default) | `quasar build` | Invokes `cargo build-sbf` with platform-tools v1.52. Requires Agave CLI v4.1.1. |
| **Upstream** | `quasar build` | Invokes the generated `build-bpf` alias with `nightly-2026-03-27` and [`sbpf-linker`](https://github.com/blueshift-gg/sbpf-linker) 0.1.9. |

Choose during `quasar init` or set in `Quasar.toml`:

```toml
[toolchain]
type = "solana"
```

### Upstream setup

The upstream toolchain requires nightly Rust, the SBF target, and the linker:

```bash
rustup toolchain install nightly-2026-03-27
rustup component add rust-src --toolchain nightly-2026-03-27
cargo install sbpf-linker --version 0.1.9 --locked
```

## Editor setup

Install [rust-analyzer](https://marketplace.visualstudio.com/items?itemName=rust-lang.rust-analyzer). Quasar programs use `#![no_std]` and `cfg(target_os = "solana")` -- the generated `Cargo.toml` configures `check-cfg` so rust-analyzer won't warn about either.
