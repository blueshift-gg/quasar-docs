## Scaffold a new project

Run `quasar init` to create a new project:

```bash
quasar init my-program
```

The CLI prompts for three choices:

1. **Toolchain** -- `solana` (default) or `upstream`. See [Installation](./installation) for details.
2. **Testing framework** -- `QuasarSVM/Rust` (default), `Mollusk`, `QuasarSVM/Web3.js`, `QuasarSVM/Kit`, or `None`.
3. **Template** -- `Minimal` (instruction file only) or `Full` (adds state and events modules).

Skip all prompts:

```bash
quasar init my-program --toolchain solana --framework quasarsvm-rust --template minimal
```

## Project structure

### Quasar.toml

Project manifest recording name, toolchain, and testing framework:

```toml
[project]
name = "my-program"

[toolchain]
type = "solana"

[testing]
framework = "quasarsvm-rust"
```

### Cargo.toml

Standard Rust crate with `crate-type = ["cdylib"]` so it compiles to a shared object the Solana runtime can load:

```toml
[package]
name = "my-program"
version = "0.1.0"
edition = "2021"

[lints.rust.unexpected_cfgs]
level = "warn"
check-cfg = [
    'cfg(target_os, values("solana"))',
]

[lib]
crate-type = ["cdylib"]

[features]
alloc = []
client = []
debug = []

[dependencies]
quasar-lang = { git = "https://github.com/blueshift-gg/quasar" }
```

Feature flags: `debug` enables verbose error logging on-chain (account name and index in constraint failures); `client` enables the off-chain instruction builder module; `alloc` enables heap allocation for programs that need it.

### src/lib.rs

The program entrypoint:

```rust
#![cfg_attr(not(test), no_std)]

use quasar_lang::prelude::*;

mod instructions;
use instructions::*;

declare_id!("<YOUR_PROGRAM_ID>");

#[program]
mod my_program {
    use super::*;

    #[instruction(discriminator = 0)]
    pub fn initialize(ctx: Ctx<Initialize>) -> Result<(), ProgramError> {
        ctx.accounts.initialize()
    }
}
```

- **`#![cfg_attr(not(test), no_std)]`** -- `no_std` on-chain, `std` during tests.
- **`declare_id!`** -- Program's on-chain address. The CLI generates a keypair in `target/deploy/` and embeds the public key here.
- **`#[program]`** -- Generates the dispatch table, entrypoint function, and (for off-chain targets) a client module.
- **`#[instruction(discriminator = 0)]`** -- Each instruction gets an explicit discriminator. Accepts a single integer (`discriminator = 0`) or a byte array (`discriminator = [1, 0, 0, 0]`). Discriminators must be unique within a program. The dispatch table matches on these bytes.
- **`Ctx<Initialize>`** -- Typed context containing the parsed and validated `Initialize` accounts struct plus PDA bumps.

### src/instructions/initialize.rs

```rust
use quasar_lang::prelude::*;

#[derive(Accounts)]
pub struct Initialize<'info> {
    pub payer: &'info mut Signer,
    pub system_program: &'info Program<System>,
}

impl<'info> Initialize<'info> {
    #[inline(always)]
    pub fn initialize(&self) -> Result<(), ProgramError> {
        Ok(())
    }
}
```

- **`#[derive(Accounts)]`** -- Generates parsing and validation. Each field is a typed account reference with automatic checks (signer, ownership, mutability).
- **`&'info mut Signer`** -- Mutable reference to a signer. `mut` = writable; `Signer` = requires signature.
- **`&'info Program<System>`** -- Reference to the System program, validated by address.
- **The `impl` block** -- Business logic. Accounts are parsed and validated before your code runs.

### Generated keypair

The CLI generates a program keypair at `target/deploy/my-program-keypair.json` and embeds the public key in `declare_id!`.

## Add a new instruction

```bash
quasar new instruction transfer
```

This creates `src/instructions/transfer.rs`, updates `src/instructions/mod.rs`, and adds the instruction to the `#[program]` block in `src/lib.rs` with the next available discriminator.

Generated file:

```rust
use quasar_lang::prelude::*;

#[derive(Accounts)]
pub struct Transfer<'info> {
    pub payer: &'info mut Signer,
    pub system_program: &'info Program<System>,
}

impl<'info> Transfer<'info> {
    #[inline(always)]
    pub fn transfer(&self) -> Result<(), ProgramError> {
        Ok(())
    }
}
```

Updated `lib.rs`:

```rust
#[program]
mod my_program {
    use super::*;

    #[instruction(discriminator = 0)]
    pub fn initialize(ctx: Ctx<Initialize>) -> Result<(), ProgramError> {
        ctx.accounts.initialize()
    }

    #[instruction(discriminator = 1)]
    pub fn transfer(ctx: Ctx<Transfer>) -> Result<(), ProgramError> {
        ctx.accounts.transfer()
    }
}
```

The discriminator auto-increments from the highest existing value in the `#[program]` block.

## Build the program

```bash
quasar build
```

Invokes `cargo build-sbf` (Solana toolchain) or `cargo +nightly build-bpf` (upstream) based on `Quasar.toml`. Also generates the IDL and client code in `target/`.

Output: `target/deploy/my_program.so`.

Build with debug logging (verbose constraint failure messages):

```bash
quasar build --debug
```

## Test the program

```bash
quasar test
```

Builds the program first (unless `--no-build` is passed), then runs the test suite. Rust frameworks (Mollusk, QuasarSVM/Rust) use `cargo test`; TypeScript frameworks (QuasarSVM/Web3.js, QuasarSVM/Kit) use the TypeScript test runner.

Filter by name:

```bash
quasar test --filter test_initialize
```

## Deploy the program

```bash
solana config set --url devnet
quasar deploy
```

Reads the keypair from `target/deploy/` and deploys the compiled `.so` file. Specify a custom keypair with `quasar deploy --program-keypair ./my-keypair.json`.

## The full template

The `Full` template adds `src/state.rs` and `src/events.rs`:

**src/state.rs** -- On-chain account types:

```rust
use quasar_lang::prelude::*;

#[account(discriminator = 1)]
pub struct MyAccount {
    pub authority: Address,
    pub value: u64,
}
```

**src/events.rs** -- Events emitted during execution:

```rust
use quasar_lang::prelude::*;

#[event(discriminator = 0)]
pub struct InitializeEvent {
    pub authority: Address,
}
```

For a full program with state, events, and multiple instructions, see [Build a Vault](/docs/guides/build-a-vault).
