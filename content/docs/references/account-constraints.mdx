title: Account Constraints
description: "Reference for all `#[account(...)]` constraint attributes in Quasar."
---

Account constraints are attributes on fields in a `#[derive(Accounts)]` struct. They generate validation code that runs during account parsing.

```rust
#[derive(Accounts)]
pub struct MyInstruction {
    #[account(mut, has_one(authority))]
    pub vault: Account<Vault>,
    pub authority: Signer,
}
```

## Custom Errors

`has_one`, `constraints`, and `address` support custom error codes with `@`:

```rust
#[account(
    has_one(authority) @ MyError::Unauthorized,
    constraints(vault.balance.get() > 0) @ MyError::EmptyVault,
    address = EXPECTED_ADDRESS @ MyError::WrongAddress
)]
```

Without `@`, these attributes use their default errors (`HasOneMismatch`, `ConstraintViolation`, `AddressMismatch` respectively).

## General Constraints

### `mut`

Marks the account as mutable. Checks that the account was passed as writable.

| | |
|---|---|
| **Syntax** | `#[account(mut)]` |
| **Error on failure** | `ProgramError::Immutable` |
| **Required for** | Any account whose data or lamports are modified |

```rust
#[account(mut)]
pub vault: Account<Vault>,
```

### `dup`

Allows this account to share the same address as a previous account in the struct. By default, duplicate accounts are rejected. Use `dup` when intentional duplication is needed.

| | |
|---|---|
| **Syntax** | `#[account(dup)]` |
| **Effect** | Skips the duplicate-address check for this field |

```rust
#[derive(Accounts)]
pub struct SelfTransfer {
    #[account(mut)]
    pub source: Account<TokenAccount>,
    #[account(mut, dup)]
    pub destination: Account<TokenAccount>,
}
```

### `address = <expr>`

Validates that the account's address matches a constant or expression.

| | |
|---|---|
| **Syntax** | `#[account(address = EXPR)]` or `#[account(address = EXPR @ ErrorVariant)]` |
| **Error on failure** | `QuasarError::AddressMismatch` (default) or custom |

```rust
use solana_address::address;
const ADMIN: Address = address!("Admin111111111111111111111111111111111111111");

#[account(address = ADMIN)]
pub admin: Signer,
```

### `has_one(...)`

Validates that a stored address field matches another account in the struct. The account type must have a field with the same name as the referenced account.

| | |
|---|---|
| **Syntax** | `#[account(has_one(field_name, ...))]` or `#[account(has_one(field_name) @ ErrorVariant)]` |
| **Error on failure** | `QuasarError::HasOneMismatch` (default) or custom |
| **Generated check** | `account.field_name == field_name.address()` |

```rust
#[account(discriminator = 1)]
pub struct Vault {
    pub authority: Address,
    pub mint: Address,
}

#[derive(Accounts)]
pub struct Withdraw {
    #[account(mut, has_one(authority, mint))]
    pub vault: Account<Vault>,
    pub authority: Signer,
    pub mint: Account<Mint>,
}
```

List multiple stored address fields in one check. Use separate checks when they
need different custom errors:

```rust
#[account(has_one(authority), has_one(mint) @ MyError::WrongMint)]
```

### `constraints(...)`

Evaluates a boolean expression. If it returns `false`, the instruction fails.

| | |
|---|---|
| **Syntax** | `#[account(constraints(EXPR, ...))]` or `#[account(constraints(EXPR) @ ErrorVariant)]` |
| **Error on failure** | `QuasarError::ConstraintViolation` (default) or custom |

```rust
#[account(
    constraints(vault.balance.get() >= amount) @ MyError::InsufficientBalance
)]
pub vault: Account<Vault>,
```

Multiple expressions can share one check group:

```rust
#[account(
    constraints(
        clock.slot.get() > vault.unlock_slot.get(),
        vault.is_active.get()
    )
)]
```

## Initialization

### `init`

Creates a new account via System program CPI. Derives size from `T::SPACE` (or `space = <expr>`), calculates rent-exempt lamports, and writes the discriminator.

| | |
|---|---|
| **Syntax** | `#[account(init, payer = field_name)]` |
| **Requires** | `payer` field, `system_program` field in the accounts struct |
| **Error on failure** | `AccountAlreadyInitialized` if discriminator is already set |

```rust
#[derive(Accounts)]
pub struct CreateVault {
    #[account(init, payer = payer)]
    pub vault: Account<Vault>,
    #[account(mut)]
    pub payer: Signer,
    pub system_program: Program<SystemProgram>,
}
```

For PDA accounts, declare typed seeds on the account type and bind the field
with `address`:

```rust
#[account(discriminator = 1)]
#[seeds(b"vault", authority: Address)]
pub struct Vault {
    pub authority: Address,
    pub bump: u8,
}

#[account(init, payer = payer, address = Vault::seeds(authority.address()))]
pub vault: Account<Vault>,
```

### `init(idempotent)`

`init(idempotent)` creates and initializes an uninitialized account. It skips creation when the account is already initialized, then validates the existing account. Add either an `address` constraint or an account behavior group that can perform that validation.

| | |
|---|---|
| **Syntax** | `#[account(init(idempotent), payer = field_name)]` |
| **Requires** | `payer` field, `system_program` field, plus `address` or a validating behavior group |

```rust
// Profile declares #[seeds(b"profile", user: Address)].
#[account(init(idempotent), payer = payer, address = Profile::seeds(user.address()))]
pub profile: Account<Profile>,
```

### `payer = <field>`

Specifies which signer pays for account creation.

| | |
|---|---|
| **Syntax** | `#[account(init, payer = field_name)]` |
| **Auto-detection** | If omitted on `init`, looks for a field named `payer` |

### `space = <expr>`

Overrides the default account size (`T::SPACE`). Use for dynamic accounts or when you need extra space.

| | |
|---|---|
| **Syntax** | `#[account(init, payer = payer, space = EXPR)]` |
| **Default** | `T::SPACE` (discriminator + sizeof fixed fields) |

```rust
#[account(init, payer = payer, space = 8 + 32 + 4 + name.len())]
pub profile: Account<Profile>,
```

## PDA (Program Derived Address)

### `#[seeds(...)]`

Declares a typed PDA layout on a program account type:

```rust
#[account(discriminator = 1)]
#[seeds(b"vault", authority: Address, vault_id: u64)]
pub struct Vault {
    pub authority: Address,
    pub vault_id: u64,
    pub bump: u8,
}
```

| | |
|---|---|
| **Syntax** | `#[seeds(b"prefix", name: Type, ...)]` |
| **Supported parameter types** | `Address`, `u8`, `u16`, `u32`, `u64`, `[u8; N]` where `N <= 32` |
| **Limits** | 16 seeds before the bump; 32 bytes per seed |

### `address = Type::seeds(...)`

Binds an accounts field to the declared layout. It validates the canonical PDA
and stores the verified bump in `ctx.bumps.<field_name>`:

```rust
#[account(
    mut,
    address = Vault::seeds(authority.address(), vault_id)
)]
pub vault: Account<Vault>,
```

If the account type contains `bump: u8`, Quasar can read that field to verify
an existing account without searching for the bump. Save the generated bump
when initializing the account.

For a PDA without a program account type, declare a unit marker:

```rust
#[derive(Seeds)]
#[seeds(b"vault", authority: Address)]
pub struct VaultPda;

#[account(address = VaultPda::seeds(authority.address()))]
pub vault: UncheckedAccount,
```

Each typed PDA field also generates a `<field>_signer` method for CPI signing.

## Lifecycle

### `close(dest = <destination>)`

Closes the account after the instruction executes. Zeros the discriminator, drains lamports to the destination, and reassigns ownership to the System program.

| | |
|---|---|
| **Syntax** | `#[account(mut, close(dest = destination_field))]` |
| **Requires** | `mut` on this field, destination field must be writable |

```rust
#[derive(Accounts)]
pub struct CloseVault {
    #[account(mut, close(dest = authority))]
    pub vault: Account<Vault>,
    #[account(mut)]
    pub authority: Signer,
}
```

### `realloc = <expr>`

Resizes account data, adjusting lamports for rent exemption. Growing takes lamports from the payer; shrinking returns excess.

| | |
|---|---|
| **Syntax** | `#[account(mut, realloc = EXPR, realloc::payer = field_name)]` |
| **Requires** | `mut` on this field |

```rust
#[account(mut, realloc = Vault::SPACE + extra_space, realloc::payer = payer)]
pub vault: Account<Vault>,
```

### `realloc::payer = <field>`

Account that pays for (or receives excess from) reallocation.

| | |
|---|---|
| **Syntax** | `#[account(realloc::payer = field_name)]` |
| **Auto-detection** | Falls back to the `init` payer, then a field named `payer` |

## SPL Token Account Behavior

`token(...)` validates a token account's mint and authority. With `init` or `init(idempotent)`, it supplies the values used to initialize the account.

| Argument | Purpose | Required |
|----------|---------|----------|
| `mint = <field>` | Expected token mint | Yes |
| `authority = <field>` | Expected token-account authority | Yes |
| `token_program = <field>` | Token program used for initialization or explicit owner validation | For initialization |

| | |
|---|---|
| **Syntax** | `#[account(token(mint = mint_field, authority = auth_field, token_program = token_program_field))]` |

```rust
use quasar_spl::accounts::token;

#[derive(Accounts)]
pub struct InitTokenAccount {
    #[account(
        init,
        payer = payer,
        token(mint = mint, authority = authority, token_program = token_program)
    )]
    pub vault_token: InterfaceAccount<Token>,
    pub mint: InterfaceAccount<Mint>,
    pub authority: Signer,
    #[account(mut)]
    pub payer: Signer,
    pub token_program: Interface<TokenInterface>,
    pub system_program: Program<SystemProgram>,
}
```

## SPL Mint Behavior

`mint(...)` validates a mint's authority and any supplied configuration. With `init` or `init(idempotent)`, it configures the new mint.

| Argument | Purpose | Required |
|----------|---------|----------|
| `authority = <field>` | Expected mint authority | Yes |
| `decimals = <expr>` | Expected decimals; initialization defaults to `6` when omitted | No |
| `freeze_authority = None` | Assert that the mint has no freeze authority | No |
| `freeze_authority = Some(<field>)` | Expected freeze authority | No |
| `token_program = <field>` | Token program used for initialization or explicit owner validation | For initialization |

| | |
|---|---|
| **Syntax** | `#[account(mint(authority = authority_field, decimals = EXPR, freeze_authority = None, token_program = token_program_field))]` |

```rust
use quasar_spl::accounts::mint;

#[derive(Accounts)]
pub struct CreateMint {
    #[account(
        init,
        payer = payer,
        mint(
            decimals = 6,
            authority = authority,
            freeze_authority = Some(authority),
            token_program = token_program
        )
    )]
    pub mint: InterfaceAccount<Mint>,
    #[account(mut)]
    pub payer: Signer,
    pub authority: Signer,
    pub token_program: Interface<TokenInterface>,
    pub system_program: Program<SystemProgram>,
}
```

## Associated Token Account (ATA) Behavior

`associated_token(...)` derives and validates an ATA from its mint, authority, and token program. Initialization invokes the ATA program.

| Argument | Purpose | Required |
|----------|---------|----------|
| `mint = <field>` | Mint used for address derivation and token-data validation | Yes |
| `authority = <field>` | Wallet or PDA used for address derivation and validation | Yes |
| `token_program = <field>` | Token program used for derivation and initialization | For initialization and explicit interface validation |
| `system_program = <field>` | System program passed to the create CPI | For initialization |
| `ata_program = <field>` | Associated Token Program invoked by the create CPI | For initialization |

| | |
|---|---|
| **Syntax** | `#[account(associated_token(mint = mint_field, authority = authority_field, token_program = token_program_field))]` |

```rust
use quasar_spl::accounts::associated_token;

#[derive(Accounts)]
pub struct InitAta {
    #[account(
        init,
        payer = payer,
        associated_token(
            mint = mint,
            authority = owner,
            token_program = token_program,
            system_program = system_program,
            ata_program = ata_program
        )
    )]
    pub ata: InterfaceAccount<Token>,
    pub mint: InterfaceAccount<Mint>,
    pub owner: Signer,
    #[account(mut)]
    pub payer: Signer,
    pub token_program: Interface<TokenInterface>,
    pub ata_program: Program<AssociatedTokenProgram>,
    pub system_program: Program<SystemProgram>,
}
```

## Metadata Account Behavior

`metadata(...)` validates a metadata account against its program, mint, and PDA address. With `init`, it supplies the arguments for `create_metadata_accounts_v3`.

| Argument | Required for validation | Required for initialization |
|----------|-------------------------|-----------------------------|
| `program = <field>` | Yes | Yes |
| `mint = <field>` | Yes | Yes |
| `mint_authority = <field>` | No | Yes |
| `update_authority = <field>` | No | Yes |
| `system_program = <field>` | No | Yes |
| `rent = <field>` | No | Yes |
| `name = <str>` | No | Yes |
| `symbol = <str>` | No | Yes |
| `uri = <str>` | No | Yes |
| `seller_fee_basis_points = <u16>` | No | No, defaults to `0` |
| `is_mutable = <bool>` | No | No, defaults to `true` |

## Master Edition Account Behavior

`master_edition(...)` validates a master edition against its program, mint, and PDA address. With `init`, it supplies the arguments for `create_master_edition_v3`.

| Argument | Required for validation | Required for initialization |
|----------|-------------------------|-----------------------------|
| `program = <field>` | Yes | Yes |
| `mint = <field>` | Yes | Yes |
| `update_authority = <field>` | No | Yes |
| `mint_authority = <field>` | No | Yes |
| `metadata = <field>` | No | Yes |
| `token_program = <field>` | No | Yes |
| `system_program = <field>` | No | Yes |
| `rent = <field>` | No | Yes |
| `max_supply = Some(<u64>)` | No | No, defaults to `Some(0)` |

Use `max_supply = None` for unlimited printable editions. A limited master edition uses `Some(n)`, and `Some(0)` creates a unique 1/1 NFT. Declare the metadata field before the master-edition field because Quasar initializes accounts in declaration order.

```rust
use quasar_metadata::{
    accounts::{master_edition, metadata},
    MasterEditionAccount, MetadataAccount, MetadataProgram,
};
use quasar_spl::{accounts::mint, Mint, TokenProgram};

#[derive(Accounts)]
pub struct CreateNft {
    #[account(mut)]
    pub payer: Signer,
    pub authority: Signer,
    pub metadata_program: Program<MetadataProgram>,
    pub token_program: Program<TokenProgram>,
    pub system_program: Program<SystemProgram>,
    pub rent: Sysvar<Rent>,
    #[account(
        init,
        payer = payer,
        mint(decimals = 0, authority = authority, token_program = token_program)
    )]
    pub mint: Account<Mint>,
    #[account(
        init,
        payer = payer,
        metadata(
            program = metadata_program,
            mint = mint,
            mint_authority = authority,
            update_authority = authority,
            system_program = system_program,
            rent = rent,
            name = "My NFT",
            symbol = "MNFT",
            uri = "https://example.com/nft.json",
            seller_fee_basis_points = 500,
            is_mutable = true
        )
    )]
    pub metadata: Account<MetadataAccount>,
    #[account(
        init,
        payer = payer,
        master_edition(
            program = metadata_program,
            mint = mint,
            update_authority = authority,
            mint_authority = authority,
            metadata = metadata,
            token_program = token_program,
            system_program = system_program,
            rent = rent,
            max_supply = Some(0)
        )
    )]
    pub master_edition: Account<MasterEditionAccount>,
}
```

## Quick reference

| Attribute | Syntax | Default Error |
|-----------|--------|---------------|
| `mut` | `#[account(mut)]` | `Immutable` |
| `dup` | `#[account(dup)]` | -- |
| `address` | `#[account(address = EXPR)]` | `AddressMismatch` |
| `has_one` | `#[account(has_one(field, ...))]` | `HasOneMismatch` |
| `constraints` | `#[account(constraints(EXPR, ...))]` | `ConstraintViolation` |
| `init` | `#[account(init, payer = p)]` | `AccountAlreadyInitialized` |
| `init(idempotent)` | `#[account(init(idempotent), payer = p)]` | -- |
| `payer` | `#[account(payer = field)]` | -- |
| `space` | `#[account(space = EXPR)]` | -- |
| typed PDA | `#[seeds(...)]` plus `#[account(address = Type::seeds(...))]` | `InvalidPda` |
| `close` | `#[account(close(dest = field))]` | -- |
| `realloc` | `#[account(realloc = EXPR)]` | `InvalidRealloc` |
| `realloc::payer` | `#[account(realloc::payer = field)]` | -- |
| `token(...)` | `#[account(token(mint = mint, authority = authority, token_program = token_program))]` | -- |
| `mint(...)` | `#[account(mint(authority = authority, decimals = 6, token_program = token_program))]` | -- |
| `associated_token(...)` | `#[account(associated_token(mint = mint, authority = authority, token_program = token_program))]` | -- |
| `metadata(...)` | `#[account(metadata(program = metadata_program, mint = mint))]` | -- |
| `master_edition(...)` | `#[account(master_edition(program = metadata_program, mint = mint))]` | -- |
