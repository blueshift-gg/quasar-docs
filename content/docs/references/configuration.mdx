Quasar uses two configuration files: a per-project `Quasar.toml` at the project root, and a global `~/.quasar/config.toml` for user preferences.

## Project Configuration (`Quasar.toml`)

Created by `quasar init` and read by `quasar build`, `quasar test`, and other project-aware commands.

### Full Schema

```toml
[project]
name = "my-program"

[toolchain]
type = "solana"

[testing]
language = "rust"

[testing.rust]
framework = "mollusk"

[testing.rust.test]
program = "cargo"
args = ["test", "tests::"]

[clients]
path = "target/client"
languages = []
```

### `[project]`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | yes | The program name. Used as the crate name, binary name, and for keypair lookup in `target/deploy/`. Hyphens are converted to underscores for the Rust module name. |

### `[toolchain]`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | yes | Build toolchain to use. |

Valid values:

| Value | Description |
|-------|-------------|
| `"solana"` | Uses `cargo build-sbf` from the Solana SDK. The standard, stable toolchain. |
| `"upstream"` | Uses `cargo build-bpf` (alias for `cargo build --release --target bpfel-unknown-none`) with `sbpf-linker` from [blueshift-gg/sbpf-linker](https://github.com/blueshift-gg/sbpf-linker). Produces smaller binaries and gives access to nightly Rust features. Requires `sbpf-linker` on PATH (install via `cargo install-with-gallery`). |

### `[testing]`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `language` | string | yes | Test harness language: `none`, `rust`, or `typescript`. |

Use `language = "none"` without a nested testing table when the project has no tests.

For Rust tests, add the framework and command Quasar should execute:

```toml
[testing]
language = "rust"

[testing.rust]
framework = "quasar-svm" # or "mollusk"

[testing.rust.test]
program = "cargo"
args = ["test", "tests::"]
```

For TypeScript tests, select the SDK and record the package-manager commands:

```toml
[testing]
language = "typescript"

[testing.typescript]
framework = "quasar-svm"
sdk = "kit" # or "web3.js"

[testing.typescript.install]
program = "pnpm"
args = ["install"]

[testing.typescript.test]
program = "pnpm"
args = ["test"]
```

`quasar test` runs the configured Rust test command when `language = "rust"`. For TypeScript, it runs the install command when `node_modules` is absent, then runs the configured test command. Test filters are inserted as Cargo filters for Rust and as `-t <PATTERN>` for other test programs.

### `[clients]`

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `path` | string | yes | Base directory for generated clients. The scaffold uses `target/client`. |
| `languages` | string array | yes | Additional clients generated by `quasar build`: `typescript`, `python`, `golang`, or `c`. |

The standalone Rust client is always generated under `<path>/rust/`, so it is not listed in `languages`. A TypeScript test project includes `"typescript"` automatically.

### Example Configurations

**Minimal project with Solana toolchain and no tests:**

```toml
[project]
name = "my-program"

[toolchain]
type = "solana"

[testing]
language = "none"

[clients]
path = "target/client"
languages = []
```

**Full project with upstream toolchain and Rust tests:**

```toml
[project]
name = "my-defi-protocol"

[toolchain]
type = "upstream"

[testing]
language = "rust"

[testing.rust]
framework = "quasar-svm"

[testing.rust.test]
program = "cargo"
args = ["test", "tests::"]

[clients]
path = "target/client"
languages = []
```

## Global Configuration (`~/.quasar/config.toml`)

User preferences across all projects. Created when you run `quasar init` or `quasar config set`.

**Location:** `~/.quasar/config.toml`

### Full Schema

```toml
[defaults]
toolchain = "solana"
test_language = "rust"
rust_framework = "mollusk"
# ts_sdk = "kit" # Used when test_language = "typescript"
template = "minimal"

[ui]
animation = true
color = true
timing = true
```

### `[defaults]`

Saved preferences used by `quasar init --yes` when an explicit flag does not override them.

| Field | Type | Valid Values | Default |
|-------|------|-------------|---------|
| `toolchain` | string (optional) | `"solana"`, `"upstream"` | not set |
| `test_language` | string (optional) | `"none"`, `"rust"`, `"typescript"` | not set |
| `rust_framework` | string (optional) | `"quasar-svm"`, `"mollusk"` | not set |
| `ts_sdk` | string (optional) | `"kit"`, `"web3.js"` | not set |
| `template` | string (optional) | `"minimal"`, `"full"` | not set |

When a default is not set, `quasar init -y` uses the framework's built-in default.

### `[ui]`

Controls CLI output behavior.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `animation` | bool | `true` | Show the animated banner on `quasar init`. Automatically set to `false` after first run. |
| `color` | bool | `true` | Use colored terminal output. |
| `timing` | bool | `true` | Show build timing information (e.g., "Build complete in 1.2s"). |

## Precedence

1. **Project config (`Quasar.toml`)** is authoritative for build and test behavior.
2. **Global config (`~/.quasar/config.toml`)** provides defaults for `quasar init` and UI preferences. Global defaults do not override project settings.

## Managing configuration

### `quasar config`

Without a subcommand, opens an interactive menu.

### `quasar config get <KEY>`

Read a config value.

```bash
$ quasar config get ui.animation
true

$ quasar config get defaults.toolchain
solana
```

### `quasar config set <KEY> <VALUE>`

Write a config value. Invalid values are rejected with valid options shown.

```bash
$ quasar config set ui.color false
  ui.color = false

$ quasar config set defaults.test_language rust
  defaults.test_language = rust

$ quasar config set defaults.rust_framework quasar-svm
  defaults.rust_framework = quasar-svm

$ quasar config set defaults.ts_sdk kit
  defaults.ts_sdk = kit

$ quasar config set defaults.toolchain invalid
  invalid value for defaults.toolchain: invalid
  valid: solana, upstream
```

Boolean values accept: `true`, `false`, `1`, `0`, `yes`, `no`, `on`, `off`.

To unset a default, use `none`, `null`, or an empty string:

```bash
$ quasar config set defaults.toolchain none
```

### `quasar config list`

Print all config values and the file path.

```bash
$ quasar config list
  config: /Users/you/.quasar/config.toml

  [defaults]
    toolchain  = solana
    framework  = mollusk
    template   = minimal

  [ui]
    animation  = false
    color      = true
    timing     = true
```

### `quasar config reset`

Restore factory defaults. Preserves `animation = false` if previously disabled.

```bash
$ quasar config reset
  config reset to defaults
```

## All config keys

| Key | Scope | Type | Valid Values |
|-----|-------|------|-------------|
| `defaults.toolchain` | Global | string | `solana`, `upstream` |
| `defaults.test_language` | Global | string | `none`, `rust`, `typescript` |
| `defaults.rust_framework` | Global | string | `quasar-svm`, `mollusk` |
| `defaults.ts_sdk` | Global | string | `kit`, `web3.js` |
| `defaults.template` | Global | string | `minimal`, `full` |
| `ui.animation` | Global | bool | `true`, `false` |
| `ui.color` | Global | bool | `true`, `false` |
| `ui.timing` | Global | bool | `true`, `false` |
| `project.name` | Project | string | any valid crate name |
| `toolchain.type` | Project | string | `solana`, `upstream` |
| `testing.language` | Project | string | `none`, `rust`, `typescript` |
| `testing.rust.framework` | Project | string | `quasar-svm`, `mollusk` |
| `testing.rust.test.program` | Project | string | Test executable, generated as `cargo` |
| `testing.rust.test.args` | Project | string array | Test arguments, generated as `["test", "tests::"]` |
| `testing.typescript.framework` | Project | string | `quasar-svm` |
| `testing.typescript.sdk` | Project | string | `kit`, `web3.js` |
| `testing.typescript.install.program` | Project | string | Package-manager install executable |
| `testing.typescript.install.args` | Project | string array | Package-manager install arguments |
| `testing.typescript.test.program` | Project | string | Package-manager test executable |
| `testing.typescript.test.args` | Project | string array | Package-manager test arguments |
| `clients.path` | Project | string | Generated-client base directory |
| `clients.languages` | Project | string array | `typescript`, `python`, `golang`, `c` |
