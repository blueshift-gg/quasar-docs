Quasar declares a PDA seed layout once with `#[seeds(...)]`, then binds account
fields to that layout with `address = Type::seeds(...)`. The compiler checks the
number and types of dynamic seeds.

## Account-local seed layouts

Put `#[seeds(...)]` on a program account type when the PDA stores that type:

```rust
#[account(discriminator = 1, set_inner)]
#[seeds(b"escrow", maker: Address)]
pub struct Escrow {
    pub maker: Address,
    pub receive: u64,
    pub bump: u8,
}
```

The first argument is a byte-string prefix. Each remaining argument names a
typed dynamic seed. Bind the type in an accounts struct:

```rust
#[derive(Accounts)]
pub struct Make {
    #[account(mut)]
    pub maker: Signer,
    #[account(init, payer = maker, address = Escrow::seeds(maker.address()))]
    pub escrow: Account<Escrow>,
}
```

`address = Escrow::seeds(maker.address())` derives the canonical PDA, validates
the supplied account address, and records the discovered bump in
`ctx.bumps.escrow`.

Use the same address expression for an existing account:

```rust
#[derive(Accounts)]
pub struct UpdateEscrow {
    pub maker: Signer,
    #[account(mut, address = Escrow::seeds(maker.address()))]
    pub escrow: Account<Escrow>,
}
```

If the account type contains a `bump: u8` field, Quasar uses that stored value
to verify an existing account without searching for the canonical bump again.
Save `ctx.bumps.escrow` into that field when initializing the account.

## Standalone PDA markers

Use a unit struct when the PDA does not store a Quasar account type, such as a
lamport-only vault:

```rust
#[derive(Seeds)]
#[seeds(b"vault", user: Address)]
pub struct VaultPda;

#[derive(Accounts)]
pub struct Deposit {
    pub user: Signer,
    #[account(mut, address = VaultPda::seeds(user.address()))]
    pub vault: UncheckedAccount,
}
```

`#[derive(Seeds)]` is only for a standalone unit marker. Program account types
receive their typed seed implementation through `#[account]` plus `#[seeds]`.

## Supported seed types

Typed parameters support:

- `Address`
- `u8`, `u16`, `u32`, and `u64`, encoded little-endian
- `[u8; N]` where `N <= 32`

The byte-string prefix and every individual seed are limited to 32 bytes.
Quasar supports at most 16 seeds before the bump, matching the Solana runtime
limit of 17 total seeds.

```rust
#[derive(Seeds)]
#[seeds(b"position", owner: Address, market: u16, side: u8, tag: [u8; 8])]
pub struct PositionPda;
```

## The generated bumps value

Every typed PDA field adds one `u8` to the accounts struct's generated bumps
type:

```rust
// Generated for Make
#[derive(Copy, Clone)]
pub struct MakeBumps {
    pub escrow: u8,
}
```

The instruction handler receives it as `ctx.bumps`:

```rust
pub fn make_escrow(&mut self, receive: u64, bumps: &MakeBumps) -> Result<(), ProgramError> {
    self.escrow.set_inner(EscrowInner {
        maker: *self.maker.address(),
        receive,
        bump: bumps.escrow,
    });
    Ok(())
}
```

## PDA signing

For every field bound to typed seeds, `#[derive(Accounts)]` generates a
`<field>_signer` helper on the accounts struct. It rebuilds the declared seed
set and appends the verified bump:

```rust
pub fn withdraw(&self, bumps: &TakeBumps) -> Result<(), ProgramError> {
    let escrow_signer = self.escrow_signer(bumps);

    self.token_program
        .transfer(
            &self.vault,
            &self.destination,
            &self.escrow,
            self.vault.amount(),
        )
        .invoke_signed(&escrow_signer)
}
```

This keeps validation and CPI signing on the same typed seed declaration. Pass
the generated signer value to `.invoke_signed(...)`; manual `Seed` arrays remain
available when an instruction needs a seed shape that is not represented by an
accounts field.

## PDA signing flow

1. `address = Type::seeds(...)` validates the account and records its bump.
2. The handler obtains the matching signer with `self.field_signer(&ctx.bumps)`.
3. `.invoke_signed(&signer)` passes the complete seed set to the runtime.

## Compile-time PDA derivation

For PDAs with fully known seeds, Quasar also provides:

```rust
pub const fn find_program_address_const(
    seeds: &[&[u8]],
    program_id: &Address,
) -> (Address, u8)
```

This derives the address and bump in a `const` context, so both values are baked
into the program binary.
