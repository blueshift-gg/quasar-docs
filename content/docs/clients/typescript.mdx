![TypeScript Client](/images/typescript-client.webp)

Quasar generates Web3.js v3 and Kit clients from the same IDL. Each client includes instruction builders, account decoders, event parsers, PDA helpers, and the program address.

## Generate a client

```bash
# Build the program and generate every language in [clients].languages.
quasar build

# Regenerate the IDL without compiling the SBF program.
quasar idl .

# Generate TypeScript from an existing IDL.
quasar client target/idl/my_program.json --lang typescript
```

The generated files live under the program name:

```text
target/client/typescript/<program-name>/
  web3.ts
  kit.ts
  package.json
```

Quasar replaces these files when it regenerates the client. Keep application code outside this directory.

## Web3.js v3

The Web3.js client uses `Address` and `TransactionInstruction` from `@solana/web3.js`:

```typescript
import { Address } from "@solana/web3.js";
import { QuasarEscrowClient } from "../target/client/typescript/quasar-escrow/web3.js";

const client = new QuasarEscrowClient();

const instruction = await client.createMakeInstruction({
  maker,
  mintA,
  mintB,
  makerTaA,
  makerTaB,
  vaultTaA,
  deposit: 1_000_000n,
  receive: 500_000n,
});

const programAddress: Address = QuasarEscrowClient.programId;
```

Builders accept one named input object. Quasar fills constant accounts and derives IDL-backed PDAs. A builder returns a promise when account resolution requires a PDA or account data.

## Kit

The Kit client exposes the same program surface with Kit instruction and address types:

```typescript
import { PROGRAM_ADDRESS, QuasarEscrowClient } from "../target/client/typescript/quasar-escrow/kit.js";

const client = new QuasarEscrowClient();

const instruction = await client.createMakeInstruction({
  maker,
  mintA,
  mintB,
  makerTaA,
  makerTaB,
  vaultTaA,
  deposit: 1_000_000n,
  receive: 500_000n,
});

console.log(PROGRAM_ADDRESS);
```

Choose the client that matches the SDK used by the rest of your application. Do not mix Web3.js and Kit address or instruction types in one call.

## Decode an account

Every `#[account]` type has a strict decoder. Pass the raw account bytes, including the discriminator:

```typescript
const client = new QuasarEscrowClient();
const escrow = client.decodeEscrow(accountInfo.data);

console.log(escrow.maker);
console.log(escrow.receive);
```

The decoder rejects the wrong discriminator, truncated data, and trailing bytes.

## Derive a PDA

Quasar emits one helper for each unique IDL seed pattern:

```typescript
import { findEscrowAddress } from "../target/client/typescript/quasar-escrow/web3.js";

const escrowAddress = await findEscrowAddress(maker);
```

The helper returns the derived address. Instruction builders call the same derivation logic when a PDA account can be resolved from their inputs.

## Type mapping

| Rust | TypeScript |
| --- | --- |
| `u8`, `u16`, `u32`, `i8`, `i16`, `i32` | `number` |
| `u64`, `u128`, `i64`, `i128` | `bigint` |
| `bool` | `boolean` |
| `Address` | `Address` |
| `[T; N]` | fixed-length tuple |
| `Vec<T>` | `T[]` |

See [QuasarSVM for TypeScript](/docs/testing/quasarsvm-typescript) to execute generated instructions without starting a validator.
