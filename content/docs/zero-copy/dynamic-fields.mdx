---
title: Dynamic Fields
description: Variable-length strings and vectors stored inline in account data, zero-copy.
---

Quasar stores variable-length account data with bounded `String<MAX>` and `Vec<T, MAX>` fields. The account holds only the current payload plus a length prefix for each field, so reads do not allocate or deserialize into heap collections.

## String and Vec

```rust
#[account(discriminator = 1, set_inner)]
pub struct MultisigConfig {
    pub creator: Address,
    pub threshold: u8,
    pub bump: u8,
    pub label: String<32>,         // max 32 bytes
    pub signers: Vec<Address, 10>, // max 10 elements
}
```

Dynamic account structs do not carry a lifetime. The generated account wrapper borrows the account buffer and exposes `&str` and `&[Address]` views through accessor methods.

The compact layout places fixed fields and every length prefix in the header, followed by the dynamic payloads:

```text
Offset     Field
------     -----
0          discriminator (0x01)
1          creator: Address (32 bytes)
33         threshold: u8
34         bump: u8
35         label byte length (u8 LE)
36         signers element count (u16 LE)
38         label UTF-8 data (0..32 bytes)
38+L       signers data (0..320 bytes)
```

`String<N>` uses a one-byte prefix by default. `Vec<T, N>` uses a two-byte prefix by default. String lengths count UTF-8 bytes; vector lengths count elements. Vector elements must satisfy Quasar's zero-copy element requirements.

The macro exposes the bounds as `MIN_SPACE` and `MAX_SPACE`. For this account, `MIN_SPACE` is 38 bytes and `MAX_SPACE` is 390 bytes.

## Reading dynamic fields

The macro generates borrowed accessors:

```rust
let label: &str = self.config.label();
let signers: &[Address] = self.config.signers();
```

## Initializing dynamic fields

Add `set_inner` to the `#[account]` attribute when you want the generated initializer. It accepts a generated `MultisigConfigInner` value, the realloc payer, and the raw rent parameters:

```rust
self.config.set_inner(
    MultisigConfigInner {
        creator: *self.creator.address(),
        threshold,
        bump: bumps.config,
        label: "",
        signers,
    },
    self.creator.to_account_view(),
    self.rent.lamports_per_byte(),
    self.rent.exemption_threshold_raw(),
)?;
```

`set_inner` computes the exact compact size, adjusts the account's rent-exempt balance through the payer, resizes the data, and writes the fixed header and dynamic payloads. A value beyond its declared capacity returns `QuasarError::DynamicFieldTooLong`.

## Mutating dynamic fields

Use `as_mut(payer)` to load a mutation guard for an existing dynamic account:

```rust
let mut guard = self.config.as_mut(self.creator.to_account_view());
if !guard.label.set("new label") {
    return Err(ProgramError::InvalidInstructionData);
}
if !guard.signers.set_from_slice(updated_signers) {
    return Err(ProgramError::InvalidInstructionData);
}
```

The guard exposes bounded `PodString` and `PodVec` values. Their mutation methods report capacity failures as `false`; the `try_*` variants return a `ZeroPodError` instead. The guard saves on drop, recomputes the exact account size, moves lamports to or from the payer to preserve rent exemption, and resizes the account when the payload length changes. Call `guard.save()?` when the updated bytes must be visible before the guard leaves scope, such as before a CPI.

## Ordering rules

Fixed fields must come before all dynamic fields. Quasar rejects a fixed field after a `String` or `Vec` because compact payloads occupy the tail of the account.

Nested dynamic fields such as `Vec<String<32>, 10>` are not supported. Store a flat zero-copy element type or split the nested data into another account.

Borrowed `&str` and `&[T]` values are supported as bounded instruction arguments with `#[max(N)]`; they are not account field types. Account data must use `String<N>` or `Vec<T, N>`.

## Prefix widths

The optional final generic selects the prefix type or its width in bytes:

```rust
pub name: String<4096, u16>,       // 2-byte prefix, max 4096 bytes
pub items: Vec<Address, 10, u8>,   // 1-byte prefix, max 10 elements
```

Accepted prefix types are `u8`, `u16`, `u32`, and `u64`; the equivalent widths are `1`, `2`, `4`, and `8`. The declared capacity must fit in the selected prefix.
