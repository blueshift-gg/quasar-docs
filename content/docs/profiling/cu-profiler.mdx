---
title: CU Profiler
description: Track static sBPF cost and deploy binary size with checked-in budgets.
---

`quasar profile` reads an unstripped sBPF binary, resolves its call graph, and
assigns each function a static instruction cost. Use it to catch compiler and
code-generation regressions before they reach runtime tests.

## Profile a program

```bash
quasar profile
quasar profile target/deploy/debug/my_program.so.debug
```

With no path, Quasar builds a debug artifact and profiles it. A stripped ELF
cannot provide function attribution. Pass the `.so.debug` file under
`target/deploy/debug/` when selecting an artifact yourself.

The terminal report lists the highest-cost functions:

```text
  my_program  2,816 CU
     1,200  42.6%  Initialize::verify
       800  28.4%  Deposit::process
       816  29.0%  +3 more (--expand)
```

Use `--expand` for the complete table. Quasar stores snapshots under
`target/profile/` and starts a flamegraph viewer on `127.0.0.1:7777`.

## Check in a budget

Generate `quasar-budget.toml` after reviewing a representative release build:

```bash
quasar profile --write-budget
```

Quasar adds five percent headroom by default. Choose another percentage with
`--headroom`:

```bash
quasar profile --write-budget --headroom 10
```

The file contains one table per program:

```toml
version = 1

[program.my_program]
binary_size = 59430
total_cu = 2957

[program.my_program.functions]
"Initialize::verify" = 1260
"Deposit::process" = 840
```

`binary_size` measures the stripped deploy ELF. Function and total ceilings use
the static profile. Quasar preserves other program tables when it updates one
entry.

Review the generated values before committing them. Headroom absorbs small
codegen movement; it should not hide a regression you have not explained.

## Enforce the budget

```bash
quasar profile --assert-budget
```

Quasar evaluates every recorded ceiling and reports all violations in stable
order. It exits with code 2 when the binary, total, or a named function exceeds
its maximum. Missing, malformed, and unsupported budget files fail instead of
creating a new baseline.

CI can consume deterministic JSON:

```bash
quasar profile --assert-budget --json > profile.json
```

`--json` does not start the flamegraph server. The report includes the measured
binary size, static total, function costs, budget status, and each violation.

## Read the number correctly

The profile total estimates the sBPF instructions reached through the resolved
call graph. It does not reproduce transaction compute consumption. Solana also
charges for syscalls and runtime work, and a transaction may take one branch of
the graph.

Use the profile budget for code and binary drift. Use QuasarSVM or validator
tests to measure runtime CU for concrete instructions and account inputs.

## Other modes

- `--watch` rebuilds after changes under `src/`.
- `--diff <PROGRAM>` opens the stored snapshot comparison.
- `--share` uploads the profile to a public GitHub Gist.

These interactive modes cannot run with budget assertions or JSON output.
