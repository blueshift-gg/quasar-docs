---
title: QuasarSVM (TypeScript)
description: Test Quasar programs locally with Vitest, Web3.js v3, or Kit.
---

QuasarSVM loads a compiled program into a local Solana VM. Tests run in-process, without a validator or RPC endpoint.

## Start from the generated test

Run `quasar init` and choose either **TypeScript (Web3.js)** or **TypeScript (Kit)**. The generated project includes:

- a Vitest test under `tests/`
- the selected package manager commands in `Quasar.toml`
- Web3.js v3 or Kit dependencies
- `@blueshift-gg/quasar-svm` pinned to `0.1.13`

The Web3.js package requires Node.js 22.13 or newer.

Build and run the generated test:

```bash
quasar test
```

`quasar test` builds the program, generates its IDL and clients, installs missing JavaScript dependencies, and runs the configured Vitest command.

## Web3.js v3 test

This is the core shape of the generated Web3.js test:

```typescript
import { Buffer } from "node:buffer";
import { Address, Keypair, TransactionInstruction } from "@solana/web3.js";
import { QuasarSvm, createKeyedSystemAccount } from "@blueshift-gg/quasar-svm/web3.js";
import { readFile } from "node:fs/promises";
import { describe, expect, it } from "vitest";

describe.concurrent("MyProgram", async () => {
  it("initializes", async () => {
    const idl = JSON.parse(
      await readFile("target/idl/my_program.json", "utf8"),
    ) as { address: string };

    const programAddress = new Address(idl.address);
    const vm = new QuasarSvm();
    vm.addProgram(
      programAddress,
      await readFile("target/deploy/my_program.so"),
    );

    const { publicKey: payer } = await Keypair.generate();
    const instruction = new TransactionInstruction({
      programId: programAddress,
      keys: [{ pubkey: payer, isSigner: true, isWritable: false }],
      data: Buffer.from([0]),
    });

    const result = vm.processInstruction(instruction, [
      createKeyedSystemAccount(payer),
    ]);

    expect(
      result.status.ok,
      `initialize failed:\n${result.logs.join("\n")}`,
    ).toBe(true);
  });
});
```

For application tests, replace the manual `TransactionInstruction` with the generated client:

```typescript
const client = new MyProgramClient();
const instruction = client.createInitializeInstruction({ payer });
```

Use `await` when a generated builder derives a PDA or resolves account data.

## Kit test

The Kit variant changes the SDK types, not the VM behavior:

```typescript
import {
  AccountRole,
  address,
  generateKeyPairSigner,
} from "@solana/kit";
import { QuasarSvm, createKeyedSystemAccount } from "@blueshift-gg/quasar-svm/kit";
import { readFile } from "node:fs/promises";
import { describe, expect, it } from "vitest";

describe.concurrent("MyProgram", async () => {
  it("initializes", async () => {
    const idl = JSON.parse(
      await readFile("target/idl/my_program.json", "utf8"),
    ) as { address: string };

    const programAddress = address(idl.address);
    const vm = new QuasarSvm();
    vm.addProgram(
      programAddress,
      await readFile("target/deploy/my_program.so"),
    );

    const payer = await generateKeyPairSigner();
    const instruction = {
      programAddress,
      accounts: [
        { address: payer.address, role: AccountRole.READONLY_SIGNER },
      ],
      data: Uint8Array.from([0]),
    };

    const result = vm.processInstruction(instruction, [
      createKeyedSystemAccount(payer.address),
    ]);

    expect(
      result.status.ok,
      `initialize failed:\n${result.logs.join("\n")}`,
    ).toBe(true);
  });
});
```

## Inspect the result

`processInstruction` returns the execution status, logs, compute units, return data, and post-execution accounts.

```typescript
const result = vm.processInstruction(instruction, accounts);

result.assertSuccess();
console.log(result.computeUnitsConsumed);

const updated = result.account(accountAddress);
if (!updated) throw new Error("expected account was not returned");
```

Pass a Solana decoder to `account` when you want decoded data:

```typescript
const token = result.account(tokenAddress, getTokenDecoder());
expect(token?.amount).toBe(1_000n);
```

## Run focused tests

Use Quasar's filter to forward a test-name pattern to Vitest:

```bash
quasar test --filter initialize
```

Run the configured package script directly when the program and generated client are already current:

```bash
npm test
```

Use `quasar test --watch` to rebuild and rerun tests when source files change.
